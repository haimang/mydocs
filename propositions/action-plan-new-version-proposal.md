## 0. TL;DR / 一句话结论(v0.2 已按业主反馈重定向)

**v0.1 走偏了:它想用「规模档位 + 前置自检闸」来治理厚薄。业主否决了这个方向——那是给 agent 加负担的 meta 机制。v0.2 改为业主要的路子:不加档位、不加前置体检,而是把两块高价内容「细化」到 agent 一看就会填的程度。**

- **保留的实证结论(§2/§3)**:行数是错误的质量指标;抽样无一份病态欠规格;薄多是工作有界(wire/gate/复用)而非偷工。所以**确实不需要"普遍加厚"或"按体量分档"**——这反过来印证了业主"别加档位"的判断。
- **业主的两条正向指令(§1)**:
  1. **reference-anchor 直接植入 AP**(不再是 v0.1 的"条件折叠/有独立文件就引用")——锚区**永远在 AP 内**,这样 agent 实现时 0 跳转、grounding 0 泄漏(正堵 MIX5)。
  2. **每份 AP 自带测试台账**:逐测试项写清——**验证什么 / 短途·spike·mega·soak / 在哪一层 / 是新增还是沿用既有用例 / 映射到哪个工作项与收口目标 / 怎么算 PASS**。过去 AP 对测试要求一直不细,这是最大的高价缺口。
- **业主否决的两条**:① 继续用 **flavor** 把 AP 复杂化(❌ 不做 S/M/L/XL 档位);② **前置体量检测**(❌ 不做开工前"这活该多大"的判定闸)——都增加 agent 负担。
- **方法论转向**:**把高价内容细化成 schema + 填好的样例**,让 agent 靠"照着填"就输出我们要的东西,而不是靠"先过一道检测闸"。**内容细化 > meta 闸门。**

**一句话**:AP 仍是**一个**模板,不分档、不前置体检;v2 只把 **reference-anchor 锚区**和**测试台账**两块做成详细、带样例的固定段,让 agent 直接理解并产出。

## 1. 业主反馈(修订驱动 · 权威立场)

> 业主于 2026-05-31 对 v0.1 的纠正,修饰后作为本次修订的权威依据。

- **指令一 · reference-anchor 直接植入**:不要"条件折叠"。**reference-anchor 应直接植入到 AP 内**,成为固定段。
- **指令二 · AP 自带测试台账**:AP 必须对"本 AP 如何收口,尤其如何测试"说清楚——**有哪些测试项、是短途 / spike / 还是 mega、测试在哪个 layer、是新增还是复用之前的测试用例**。过去 AP 内一直没把测试部分详细要求;**今后每个 AP 自带自己的测试清单,要有一个明确的测试台账**。
- **否决一 · 不再用 flavor 复杂化 AP**:拒绝把 action-plan 继续通过 flavor(档位/变体)复杂化——**会加大 agent 负担**。
- **否决二 · 不做前置体量检测**:拒绝开工前的体量/规模判定——同样加负担。
- **正向方法**:**把高价的内容细化**,让 agent 更好地理解、并输出我们想要的内容。

> **业主一句话**:别给 agent 加门槛和分类,把真正值钱的部分(grounding 锚 + 测试台账)讲细、给样例,让它照着产出。

## 3. 厚薄解剖 + 诚实修正(为什么不需要档位/前置闸)

> `eval-state-analysis §3 对账诚实`:逐份核对,"薄=偷工"前提不成立。

- **薄是工作有界,不是欠规格**(子代理 C):MIX4/5/6、MCX6×2、BN0/3a/4N 全部结构完整、§9 含真实执行日志;薄因"wire-only / gate / 复用既有"。唯一 12 行文件是放错的运维便签。薄 ✗↔ 轨道(MIX6 重 290 ≈ BN4N 轻 297)。
- **唯一真缝 = MIX5 的 under-grounding 泄漏**:安全任务但威胁模型既没在上游做、又没在 AP 写够。**这正是"reference-anchor 直接植入"要堵的**(§5)。
- **由此推出**:既然薄不是病,就**不该用前置体量检测去"抓薄"**——业主否决得对。真正缺的不是"控制厚薄的闸",是**两块高价内容本身不够细**:grounding 锚 + 测试要求。**v2 = 细化这两块,不加闸。**

## 9. Tradeoff 辩证

**选"细化两块高价固定段",而非"加档位/前置闸"(v0.1)或"放任现状"**
- **为什么**:业主反馈 + 实证(薄不是病)共同指向——质量缺口在**内容不够细**(grounding 锚、测试要求),不在"缺管控闸"。细化 schema + 样例让 agent 照填,负担更低、产出更准。
- **代价**:① 每份 AP 多两块固定段(reference-anchor 锚区 + 测试台账)——但都是**本就该有的高价内容**,非仪式;② 锚区与独立 reference-anchor 可能少量重叠(摘录),靠 §7.3 指回真源控制双源漂移。
- **重评条件**:若测试台账被填成走形式(只列名不给四元组/不分层)→ 需在 §8 加更硬的样例约束或轻量 lint(但仍不前置、只在 review 期)。
